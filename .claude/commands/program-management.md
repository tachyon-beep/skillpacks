# Using Program Management

<!-- Content authority lives in plugins/axiom-program-management/skills/using-program-management/SKILL.md. Do not add inline router content to this command. -->

## Overview

Program management here means managing the delivery of work, not building it. It runs a project or program: flow, scope, forecast, dependencies, risks, status, stakeholders, and outcomes, while routing code and architecture work elsewhere.

## When to Use

Use this wrapper when delivery is busy but outcomes are not landing, "when will it be done" lacks a defensible answer, dependencies keep surprising the team, status reporting is unreliable, or several projects must be coordinated toward one outcome.

Do not use this for writing code, picking an architecture, or designing a single feature.
