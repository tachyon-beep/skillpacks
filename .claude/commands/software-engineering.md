# Software Engineering Foundations Routing

**Engineering excellence is methodology, not heroics. Systematic approaches beat clever improvisation. This pack covers process and discipline; language-specific knowledge lives in the language packs.**

Use the `using-software-engineering` skill from the `axiom-engineering-foundations` plugin to route to the right specialist sheet. Content authority lives in `plugins/axiom-engineering-foundations/skills/using-software-engineering/SKILL.md` - this wrapper is a thin pointer.

## Sheets

- **complex-debugging** - scientific method for horrible bugs: 6 phases, heisenbug protocol, investigation reset protocol, correlation-ID tracing, cognitive-bias warnings
- **systematic-refactoring** - safe incremental code transformation: characterize then plan then small steps then verify then commit; strangler-fig, parallel-change, branch-by-abstraction
- **code-review-methodology** - systematic review process: context then high-level then correctness then quality; comment categories, actionable-feedback patterns
- **incident-response** - production-fire methodology: assess then contain then mitigate then diagnose then fix then postmortem; rollback-first discipline, severity matrix
- **technical-debt-triage** - identification, categorization, prioritization: impact/cost scoring, ROI, debt register, pay-down vs live-with
- **codebase-confidence-building** - internalize systems you maintain: orient then trace then interrogate then experiment then document; distinct from architecture archaeology
