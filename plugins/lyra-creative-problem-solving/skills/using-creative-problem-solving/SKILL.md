# Using Creative Problem-Solving

## Overview

This meta-skill routes you to the right creative problem-solving technique based on your situation. Load this skill when you need creative thinking but aren't sure which approach to use.

**Core Principle**: Different creative challenges require different techniques. Match your problem type to the appropriate methodology, apply systematically.

## When to Use

Load this skill when:
- Facing a difficult or novel problem requiring creative solutions
- User mentions: "brainstorm", "creative", "innovative", "think outside the box", "new approach", "fresh perspective"
- Stuck on a problem and standard approaches aren't working
- Need to generate multiple solution options
- Evaluating unconventional ideas or alternatives

**Don't use for**: Straightforward implementation tasks, well-defined algorithmic problems, pure optimization tasks

## Routing by Problem Type

### Structured Innovation Process

**Symptoms**: "Need a systematic approach", "How to run an innovation workshop", "Step-by-step creative process", "Design new product/feature"

**Route to**: `design-thinking-methodology`

**Why**: Design thinking provides a structured 5-phase framework (Empathize, Define, Ideate, Prototype, Test) for systematic innovation. Use when you need a complete problem-solving process from understanding to validation.

**Examples**:
- "Design a new user onboarding experience" → design-thinking-methodology
- "How to structure an innovation sprint?" → design-thinking-methodology
- "Need systematic approach to product development" → design-thinking-methodology

---

### Breaking Mental Blocks

**Symptoms**: "Stuck thinking the same way", "Need different perspective", "Challenge assumptions", "Think laterally", "Unconventional solutions"

**Route to**: `lateral-thinking-techniques`

**Why**: Lateral thinking provides specific techniques (provocation, random entry, challenge assumptions) to break habitual thought patterns and discover non-obvious solutions.

**Examples**:
- "Keep solving this problem the same way" → lateral-thinking-techniques
- "How to think about this differently?" → lateral-thinking-techniques
- "Challenge core assumptions" → lateral-thinking-techniques

---

### Generating Many Ideas

**Symptoms**: "Brainstorm session", "Generate ideas", "Need 20 options", "Ideation", "Team brainstorming"

**Route to**: `brainstorming-facilitation`

**Why**: Effective brainstorming requires specific facilitation techniques to maximize idea generation and avoid common pitfalls (premature judgment, groupthink, dominant voices).

**Examples**:
- "Run a brainstorming session for our team" → brainstorming-facilitation
- "How to facilitate ideation workshop?" → brainstorming-facilitation
- "Generate 50 feature ideas" → brainstorming-facilitation

---

### Working with Constraints

**Symptoms**: "Limited resources", "Tight constraints", "Work within limitations", "Use constraints creatively", "Embrace limitations"

**Route to**: `constraint-based-creativity`

**Why**: Constraints can enhance creativity when approached correctly. Learn to use limitations as creative catalysts rather than barriers.

**Examples**:
- "Only have 2 weeks and $1000 budget" → constraint-based-creativity
- "Design solution with these strict limitations" → constraint-based-creativity
- "Turn constraints into advantages" → constraint-based-creativity

### Cross-Domain Problem Solving

**Symptoms**: "How do other industries solve this?", "Apply patterns from different field", "Analogy", "What can we learn from X?", "Biomimicry", "Cross-pollination"

**Route to**: `analogical-reasoning`

**Why**: Analogical reasoning helps transfer successful patterns from one domain to another, finding inspiration in unexpected places.

**Examples**:
- "How do logistics companies solve routing? Apply to our scheduling problem" → analogical-reasoning
- "What can game design teach us about engagement?" → analogical-reasoning
- "Find nature-inspired solutions to our challenge" → analogical-reasoning

---

### Systematic Idea Transformation

**Symptoms**: "Improve existing idea", "Modify current approach", "Adapt existing solution", "Remix concepts", "SCAMPER"

**Route to**: `scamper-technique`

**Why**: SCAMPER (Substitute, Combine, Adapt, Modify, Put to other uses, Eliminate, Reverse) provides systematic prompts to transform existing ideas into new variations.

**Examples**:
- "Take existing feature and make it innovative" → scamper-technique
- "How can we adapt competitor's approach?" → scamper-technique
- "Generate variations on this concept" → scamper-technique

---

### Fundamental Rethinking

**Symptoms**: "Question everything", "Start from first principles", "Why do we do it this way?", "Ignore best practices", "Rebuild from fundamentals"

**Route to**: `first-principles-thinking`

**Why**: First principles thinking breaks problems down to fundamental truths and rebuilds from there, bypassing inherited assumptions and conventional wisdom.

**Examples**:
- "Why does authentication work this way? Can we rethink it?" → first-principles-thinking
- "Forget how it's always been done, what should we do?" → first-principles-thinking
- "Question the entire approach" → first-principles-thinking

### Choosing Between Ideas

**Symptoms**: "Evaluate ideas", "Which option is best?", "Prioritize concepts", "Test feasibility", "Compare alternatives", "Decision criteria"

**Route to**: `idea-evaluation-frameworks`

**Why**: Structured evaluation prevents choosing ideas based on charisma or recency. Use frameworks to assess impact, feasibility, novelty, and alignment.

**Examples**:
- "Have 30 ideas, need to pick top 3" → idea-evaluation-frameworks
- "How to evaluate which concept to build?" → idea-evaluation-frameworks
- "Create decision matrix for ideas" → idea-evaluation-frameworks

---

### Testing Ideas Quickly

**Symptoms**: "Validate concept", "Quick test", "Low-fidelity prototype", "MVP", "Proof of concept", "Experiment design"

**Route to**: `prototyping-strategies`

**Why**: Effective prototyping matches fidelity to questions you need answered. Learn when to use paper sketches vs clickable mockups vs code prototypes.

**Examples**:
- "Test this UX concept quickly" → prototyping-strategies
- "Build minimal version to validate" → prototyping-strategies
- "What level of prototype do I need?" → prototyping-strategies

### Idea Generation Strategy

**Symptoms**: "Generate many options vs narrow down", "Explore vs decide", "Divergent thinking vs convergent thinking", "When to stop generating ideas?"

**Route to**: `divergent-convergent-thinking`

**Why**: Understanding when to diverge (generate many options) vs converge (narrow and refine) is critical to creative process effectiveness.

**Examples**:
- "Keep narrowing too early, kill good ideas" → divergent-convergent-thinking
- "Stuck generating forever, can't decide" → divergent-convergent-thinking
- "Structure the ideation process" → divergent-convergent-thinking


## Anti-Patterns

### ❌ Jump to Solution
**Symptom**: Immediately implement first idea without exploration
**Fix**: Use design-thinking-methodology or brainstorming-facilitation to generate options

### ❌ Endless Ideation
**Symptom**: Generate ideas forever, never move to decision/action
**Fix**: Use divergent-convergent-thinking and idea-evaluation-frameworks

### ❌ Ignore Constraints
**Symptom**: Brainstorm "ideal" solutions that are impossible to implement
**Fix**: Use constraint-based-creativity to work within real limitations

### ❌ Premature Judgment
**Symptom**: Criticize ideas during generation, kill unconventional thinking
**Fix**: Use brainstorming-facilitation rules (separate generation from evaluation)

### ❌ Reinvent Everything
**Symptom**: Ignore existing patterns, start from scratch unnecessarily
**Fix**: Use analogical-reasoning to learn from others, scamper-technique to adapt

## The Bottom Line

Creative problem-solving isn't about random inspiration—it's about applying the right technique at the right time. Use this router to match your creative challenge to proven methodologies.

**When in doubt**: Start with `design-thinking-methodology` for structured innovation, or `lateral-thinking-techniques` to break mental blocks.
